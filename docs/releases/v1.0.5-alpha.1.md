# Alpha 1.0.5

Password recovery completion release (SQL + PAM) with sender readiness diagnostics.

## Included

- Makes recovery email mandatory at registration and enforces it must be different from login email.
- Adds soft recovery-email completion prompts for existing accounts missing a valid distinct recovery address.
- Completes reset operability checks with truthful capabilities and sender diagnostics (`ready`, `degraded`, `external`).
- Switches default reset sender to `no-reply@<base_domain>` with SQL auto-provisioning and PAM external-management signaling.
- Removes PAM reset dead-end on unmapped mail login by falling back to account email during reset execution.
