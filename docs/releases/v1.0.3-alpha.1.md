# Alpha 1.0.3

- Added recovery-email based password reset flow: reset requests now deliver a token to the account recovery email, then require token confirmation before password change.
- Added legacy-account recovery bootstrap: users missing a recovery email are prompted after login to set one.
- Added authenticated recovery email update API and UI wiring (`/api/v1/me/recovery-email`) with validation.
- Improved reset delivery defaults: installer now configures a dedicated recovery sender identity by default.
- Included privileged-path hardening and rollout scaffolding from this cycle (PAM reset helper and updater worker security contracts, service wiring, and regression coverage).
