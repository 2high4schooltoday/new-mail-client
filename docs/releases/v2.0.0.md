# Despatch v2.0.0

Despatch 2.0.0 upgrades the app from a minimal mail viewer to a full mail suite foundation with strong MFA and security-first architecture.

## Highlights

- Added `/api/v2` platform with expanded mail, account, search, draft, rules, security, and crypto endpoints while keeping `/api/v1` compatibility.
- Implemented MFA onboarding and enforcement end-to-end across backend and frontend:
  - registration preference (`none`, `totp`, `webauthn`)
  - required setup/verification stages
  - legacy-user one-time MFA prompt
  - trusted devices support
  - admin MFA enforcement with OOBE-first setup requirement
- Added new database migrations (`007`-`018`) for multi-account mail, indexing, rules/Sieve, MFA, crypto metadata, sync state, quota, and trusted devices.
- Introduced Rust MailSec service/contracts for parser and security-critical operations and integrated backend client wiring.
- Expanded UI with MFA flows (TOTP, WebAuthn, recovery flow), trusted-device management, and upgraded auth/setup behavior.

## Security and Reliability

- Enforced MFA stage checks on protected v1 and v2 routes.
- Added stricter auth-stage responses and session handling paths.
- Added test coverage for MFA setup/verification gating, admin MFA policy, legacy prompt behavior, and preference switching constraints.

## Notes

- This is a major release and includes schema changes; apply migrations before runtime.
- Existing `/api/v1` integrations remain supported.
