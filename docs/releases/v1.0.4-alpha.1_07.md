# Alpha 1.0.4_07

Updater hardening release for passkey reliability on Ubuntu.

## Included

- Fixes web-UI update safety so updates cannot silently leave passkeys broken when `MAILSEC_ENABLED=true`.
- Enforces mailsec prerequisites during update: required binary + systemd unit must exist (from release payload, existing install, or `/opt/despatch/deploy`).
- Adds post-update validation that `despatch-mailsec` starts and creates `MAILSEC_SOCKET`; update fails and rolls back if this check fails.
- Improves Admin Update messaging to clearly surface mailsec-related apply failures.
- Applies these safeguards in both updater implementations (`rust/update_worker` primary path and Go fallback path).
