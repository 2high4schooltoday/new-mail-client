# Alpha 1.0.7_03

Updater permissions and Admin updater-status resilience hotfix for the Alpha 1.0.7 line.

## Included

- Fixes updater directory contract drift by enforcing `request/status` as `root:despatch` `0770` and preserving `lock/work/backups` as `root:root` `0750` in both updater workers.
- Adds runtime auto-heal of updater directory mode/ownership in privileged worker paths to prevent recurring write-probe failures.
- Aligns Go updater queue/status write paths with the same directory contract to avoid recreating `request/status` with `0750` parents.
- Improves Admin web updater polling so HTTP `503` shows updater-specific diagnostics/transient recovery messaging instead of generic refresh-failure text.
- Adds regression tests in Go and Rust to lock updater directory mode behavior and prevent future contract regressions.
